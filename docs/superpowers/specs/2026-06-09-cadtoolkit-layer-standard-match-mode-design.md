# CadToolkit 图层规范匹配方式设计

## 目标

让 CadToolkit 的图层规范更可预测、更安全。

新的规则是：

- `[LayerMap]` 里的别名默认按全字匹配。
- 需要包含、前缀、后缀匹配时，必须显式写 `*` 通配符。
- 执行前预览要解释每个图层为什么会迁移、保持不动，或被归到 `0` 层。

这次变更同时覆盖项目默认配置和用户本机正在使用的 `C:\CadToolkit\CadToolkit.ini`。

## 当前行为

`LayerStandardWhitelist` 已经是“无 `*` 则全字匹配，有 `*` 则通配匹配”。

但 `[LayerMap]` 目前是混合规则：

- 标准图层名按全字匹配。
- 不含数字的别名按包含匹配。
- 含数字的别名按边界匹配。

这种方式对旧图纸比较宽容，但用户只看 INI 配置时不容易判断实际会命中哪些图层。

## 新匹配语义

`[LayerMap]` 别名改成和白名单一致的语义：

- `设备` 只匹配图层名正好叫 `设备`。
- `*设备*` 匹配包含 `设备` 的图层。
- `设备*` 匹配以 `设备` 开头的图层。
- `*设备` 匹配以 `设备` 结尾的图层。
- `0-4` 只匹配图层名正好叫 `0-4`。

匹配仍然忽略大小写。

## 默认 LayerMap

项目默认模板和内置默认配置统一改成下面这版。中文语义词使用显式通配符，数字编码和短英文代码保留全字匹配：

```ini
[LayerMap]
0-设备层=*设备*,0-4,VIS35
1-中心线层=*中心*,*中心线*,CENTER,0-1,1,AXIS,CLEARANCE
2-虚线层=*虚线*,HIDDEN,DASH,HID
3-文字层=*文字*,*说明*,*编号*,TEXT,txt
4-标注层=*标注*,*尺寸*,DIM,dim
5-风网=*风网*,*风管*,*风道*,0-5
6-溜管=*溜管*,*压运*,VIS5
7-洞孔=*洞口*,*洞孔*,*开洞*
8-受力点=*受力*,*受力点*,*吊点*
9-建筑=*建筑*,ARCH
10-非标=*非标*,*非标准*
11-填充=*填充*,HATCH,PAT
```

## 本机配置迁移

用户要求同步更新本机正在使用的 `C:\CadToolkit\CadToolkit.ini`。

迁移只替换 `[LayerMap]` 段，不修改：

- 根配置，例如 `LayerStandardFallbackTo0`
- `[Commands]`
- `[LayerStandard]`
- 其它注释和 section

写入前记录当前文件哈希。写入后确认只有 `[LayerMap]` 段按预期变化。

## 预览解释

保留现有的文本预览窗口，不做表格 UI。

预览中每个图层要显示命中原因，例如：

- `命中标准图层名`
- `命中别名 "*设备*"（通配匹配）`
- `命中别名 "0-4"（全字匹配）`
- `命中白名单 "0"（全字匹配）`
- `命中白名单 "*图框*"（通配匹配）`
- `未识别且启用归 0`

当预览窗口里勾选“未识别图层归 0”时，未识别区域要明确提示这些非白名单图层会移动到 `0` 层。

## 错误处理

未识别且未启用归 `0` 的图层保持原样，并在预览中说明“将保持原样”。

空白白名单项忽略。

如果多个别名都可能命中同一个源图层，保持现有规则顺序：先匹配到的规则生效。预览显示最终生效的别名，方便用户调整配置。

## 测试范围

增加或更新 PowerShell 测试，覆盖：

- `[LayerMap]` 普通别名只全字匹配，不命中更长图层名。
- `[LayerMap]` 通配别名可以匹配包含关系。
- `0-4` 这类数字别名默认只全字匹配，除非显式写通配符。
- 预览文本显示命中的别名和匹配方式。
- 预览文本显示命中的白名单项和匹配方式。
- 开启和关闭未识别归 `0` 时，预览文本不同。
- 项目模板、内置默认配置、本机配置都使用新的通配写法。

## 不在本轮范围

本轮不做表格 UI、配置编辑器、别名冲突检测，也不自动推断并改写任意用户自定义别名。除了已确认的本机 `[LayerMap]` 替换，其它配置保持不动。
