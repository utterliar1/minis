# BlockBrowser 手动测试清单

用于测试部署在 `C:\BlockBrowser` 的 BlockBrowser 1.3.2 版本。

建议先测试浩辰，再测试 AutoCAD 2020 和中望 2020。每个平台测试前先重启 CAD，确保加载的是最新 DLL。

## 1. 启动与基础状态

- [ ] CAD 启动后可加载 `C:\BlockBrowser\autoload.lsp`。
- [ ] 输入 `BB` 可以打开 BB 面板。
- [ ] 输入 `KLLQ` 可以打开 BB 面板。
- [ ] 输入 `BBINFO` 可以显示版本、库路径和命令列表。
- [ ] BB 面板底部显示版本号 `v1.3.2 | WLUP`。
- [ ] 设置窗口可以打开，不被 2K 屏幕 175% 缩放遮挡。
- [ ] 关闭并重新打开 BB 面板，窗口大小和缩略图大小保持正常。

## 2. 分类栏

- [ ] 分类按钮下边框完整显示，没有被遮挡。
- [ ] 分类较少时，不出现多余横向滚动条空间。
- [ ] 分类较多时，原生横向滚动条出现并可以拖动。
- [ ] 分类右侧 `+` 按钮可点击，位置和分类按钮高度协调。
- [ ] 新建分类成功后，分类列表刷新。
- [ ] 空分类可以作为“添加到库”和“导入块”的目标。
- [ ] BB 面板浏览分类不显示空分类。

## 3. 浏览与插入

- [ ] “全部”分类可以显示所有有 DWG 的块。
- [ ] “最近”分类可以显示最近插入记录。
- [ ] 搜索框可以按块名过滤。
- [ ] 切换小/中/大/特大缩略图尺寸后布局正常。
- [ ] 单击块卡片后选中状态明确。
- [ ] 双击块卡片可进入插入流程。
- [ ] 点击“插入”可进入插入流程。
- [ ] CAD 命令行提示插入点。
- [ ] 插入比例和旋转角度按设置生效。

## 4. 块库管理

- [ ] “添加到库”可选择已有分类。
- [ ] “添加到库”可选择空分类。
- [ ] 输入非法分类或块名时会提示，不会继续写入。
- [ ] 保存块时可选择对象并指定基点。
- [ ] 保存成功后块出现在对应分类。
- [ ] “导出块”可以打开导出窗口。
- [ ] 导出窗口支持多选当前图纸块定义。
- [ ] 导出到已有分类成功。
- [ ] 导出到空分类成功。
- [ ] 导出完成后命令行显示成功和失败数量。
- [ ] 重命名块成功后，卡片名称和文件名同步更新。
- [ ] 重命名为空名或非法名时提示。
- [ ] 重命名为已存在名称时提示。
- [ ] 删除块前有确认提示。
- [ ] 删除成功后卡片从列表移除。

## 5. 设置

- [ ] 设置窗口显示当前库路径。
- [ ] 修改插入比例后保存，再插入块验证比例生效。
- [ ] 修改旋转角度后保存，再插入块验证角度生效。
- [ ] 设置为空路径时提示。
- [ ] 设置为不存在路径时提示是否创建。
- [ ] 切换库路径后 BB 面板自动刷新。
- [ ] 保存设置后 `C:\BlockBrowser\config.ini` 保留并更新用户配置。

## 6. 缩略图

- [ ] 首次打开图库时缩略图逐步加载，界面不卡死。
- [ ] 已加载缩略图再次打开时能复用缓存。
- [ ] `BBTHUMB` 可以清除缩略图缓存。
- [ ] 面板「图库 > 补全缩略图」可以为当前列表缺失缓存的块生成缩略图，不修改 DWG 文件。
- [ ] 面板「图库 > 状态诊断」显示当前模式、图库路径、NAS/本地副本可访问性、本地变更记录数量和缩略图缓存数量，并可复制。
- [ ] 清除后重新打开 BB 面板，缩略图可重新生成。
- [ ] 没有预览图的 DWG 至少显示占位图，不出现空白卡片。

## 7. NAS 与本地副本

在办公室 NAS 可访问时测试：

- [ ] 面板上的“更新本地图库”可以从 NAS 更新本地图库，并同步 NAS 上的新增、修改和删除。
- [ ] “更新本地图库”执行前先显示树形预览，按新增、覆盖、删除、保护跳过分组；取消时不会修改本地文件。
- [ ] `BBMIRROR` 执行前先显示文本预览，取消时不会修改本地文件。
- [ ] `BBMIRROR` 可以从 NAS 更新本地图库，并保留有本地变更记录的文件。
- [ ] “更新本地图库”和 `BBMIRROR` 完成后显示新增、覆盖、删除、保护跳过数量。
- [ ] `ProtectedLocalCategories=个人块` 时，`BBMIRROR` 不覆盖、不删除本地 `个人块` 分类里的 DWG。
- [ ] `AllowNasSync=0` 时，BB 面板不显示“同步中心”，`BBSYNC` 会提示联系指定维护人。
- [ ] `AllowNasSync=0` 且 `CurrentLibraryMode=Auto` 时，如果本地副本存在，优先使用本地副本；如果只临时浏览 NAS，新增、删除、重命名等写入 NAS 的操作会被拦截。
- [ ] `AllowNasSync=1` 时，BB 面板显示“同步中心”，且不再显示单独的“同步到NAS”入口。
- [ ] 普通同事在本地副本删除块时，会删除本地文件，不会写 NAS；如果 NAS 仍有该块，下次更新本地图库会恢复。
- [ ] NAS 可用时，BB 面板状态显示 NAS 路径。
- [ ] `BBSYNC` 会打开同步中心；在同步中心执行同步后，可以上传本地新增且 NAS 不存在的块。
- [ ] `BBSYNC` 打开的同步中心会跳过 `ProtectedLocalCategories` 中的分类，不会自动上传 `个人块`，并显示为“白名单跳过”。
- [ ] NAS 已有同名块时，同步中心跳过并报告重复。
- [ ] 指定维护人在本地副本删除块时，只记录删除请求，不自动删除 NAS 文件。
- [ ] 同步结果显示上传、重复跳过、白名单跳过、冲突、删除待确认、失败数量。

在 NAS 不可访问时测试：

- [ ] `CurrentLibraryMode=Auto` 时可以自动使用本地副本。
- [ ] BB 面板状态显示本地副本路径或 NAS 不可用提示。
- [ ] 可以在本地副本继续添加块。
- [ ] 可以在本地副本继续重命名块。
- [ ] `AllowNasSync=0` 时可以在本地副本删除本地文件；`AllowNasSync=1` 时可以在本地副本记录删除请求。
- [ ] 回到 NAS 环境后，`BBSYNC` 打开的同步中心可以识别这些本地变更。
