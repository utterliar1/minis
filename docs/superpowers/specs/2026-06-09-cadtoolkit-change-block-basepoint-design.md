# CadToolkit 改块基点设计

## 背景

CadToolkit 现有图块操作包括重命名块、快捷建块、按块选择。用户希望增加“改块基点”功能，用于在整理图纸时调整块的插入基点。

该功能的关键要求是：修改块定义基点后，图面上已有同名块参照的视觉位置保持不变。也就是说，基点变化不应造成现有图形在图纸中漂移。

## 目标

- 新增命令 `CT_CHANGEBASEPOINT`，配置面板显示名称为“改块基点”。
- 用户选择一个块参照，再指定新的基点。
- 插件修改该块定义的基点，并反向移动块定义内对象，使所有现有同名块参照在图面上看起来不动。
- 操作应进入单次 undo，方便用户撤销。
- 同步默认配置、README 和使用手册。

## 非目标

- 不处理外部参照、布局块、匿名块。
- 动态块先保守拒绝处理，避免破坏动态块参数和可见性状态。
- 不为选中块自动生成新块名；本功能默认修改同名块定义，因此影响同名块的所有参照。
- 不提供批量改多个块基点的流程。

## 用户流程

1. 用户运行 `CT_CHANGEBASEPOINT`，或从 `CC` 面板点击“改块基点”。
2. 插件提示选择一个块参照。
3. 插件读取该参照对应的块定义，并检查是否可修改。
4. 用户指定新的基点。
5. 插件执行修改。
6. 命令行提示已修改块名，并提示同名块参照数量。

## 核心行为

选择块参照后，用户指定的新基点是世界坐标点。插件需要把该点转换到块定义坐标系中，得到新的定义原点。

修改时采用“定义坐标内反向平移”的方式：

- 旧块定义原点记为 `oldOrigin`。
- 新块定义原点记为 `newOrigin`。
- 定义坐标偏移量为 `delta = newOrigin - oldOrigin`。
- 将块定义内实体整体移动 `-delta`。
- 将块定义 `Origin` 设置为 `newOrigin`。

这样块参照在图纸中的插入位置不被逐个改写，现有同名块的图形视觉位置保持不变，而块的基点位置更新为用户指定点。

## 边界处理

- 如果用户取消选择或取消点选，命令静默退出。
- 如果选中对象不是块参照，提示只能选择块参照。
- 如果块定义不可写、来自外部参照、是布局块、是匿名块或动态块，提示当前块不支持改基点。
- 如果块定义内实体移动失败，事务不提交，保留原图状态。
- 操作完成后输出清晰摘要，例如：`已修改块 "A-DOOR" 的基点，影响 12 个同名参照。`

## 配置和文档

新增命令需要同步到：

- `CadToolkit/CadToolkit.ini`
- `CadToolkit/CadToolkit.default.ini`
- `CadToolkit/src/CadToolkit.Core/Config.cs`
- `CadToolkit/README.md`
- `CadToolkit/CadToolkit使用手册.html`

README 中“图块操作”数量从 3 个更新为 4 个。

## 测试策略

先添加测试，再实现功能。测试重点：

- `BlockCommands.cs` 包含 `CT_CHANGEBASEPOINT` 命令。
- 新命令使用块参照选择流程。
- 代码包含从世界坐标转换到块定义坐标的逻辑。
- 代码包含反向移动块定义内实体以保持现有块视觉位置的逻辑。
- 默认配置、模板配置、内置配置、README 和使用手册均包含“改块基点”与 `CT_CHANGEBASEPOINT`。
- 图块操作数量更新为 4 个。

运行验证：

- 执行 `CadToolkit/tests/*.ps1` 全量测试。
- 执行 `git diff --check`。
- 如代码实现完成后需要本地测试，先确认 CAD 进程已关闭，再运行 `CadToolkit/deploy-local.ps1`。

## 验收标准

- 运行 `CT_CHANGEBASEPOINT` 可以选择块并指定新基点。
- 普通块的基点被更新，图面上的同名块看起来不发生位移。
- 不支持的块类型有明确提示，不会修改图纸。
- `CC` 面板能通过配置显示该命令。
- 文档和默认配置都能找到该命令。
- 全量测试通过，部署脚本可以正常构建三端包。
