# CadToolkit 图层规范 TreeView 分组预览设计

## 目标

大图纸执行 `CT_LAYERSTANDARD` 时，当前纯文本预览会出现信息过长、横向滚动、重复原因太多的问题。本轮目标是把预览改成可展开/折叠的分组结构，让用户先看风险摘要，再展开查看明细。

## 范围

本轮只改图层规范的执行前预览界面和预览数据组织方式。

保留：

- 图层统计范围
- 图层匹配规则
- 白名单和未识别归 `0` 的行为
- ByLayer、删除空旧图层、执行/取消按钮
- 实际迁移和删除逻辑

新增：

- `TreeView` 分组预览
- 摘要节点
- 未识别图层节点
- 按目标图层分组的迁移节点
- 白名单图层节点
- 勾选“未识别图层归 0”时重建预览树

## UI 布局

对话框仍使用 WinForms 固定对话框，但把原来的只读 `TextBox` 替换为 `TreeView`。

建议尺寸：

- 窗口宽度保持约 `560` 逻辑像素。
- 高度可以从 `470` 增加到约 `520`，给树控件更多空间。
- `TreeView` 占据顶部主要区域。
- 三个复选框和执行/取消按钮保留在底部。

## TreeView 结构

顶层节点按风险和决策顺序排列：

```text
 摘要
 未识别图层（3 层 / 20 对象）
 将迁移图层（18 层 / 1120 对象）
   0-设备层（14 层 / 1102 对象）
   3-文字层（3 层 / 18 对象）
 白名单图层（2 层 / 8 对象）
```

### 摘要节点

摘要节点显示总体数字：

```text
摘要：标准图层 12 个；将迁移 18 层 / 1120 对象；未识别 3 层 / 20 对象；白名单 2 层 / 8 对象
```

摘要节点默认展开，下面用子节点列出：

```text
标准图层：12 个
将迁移：18 个旧图层 / 1120 个对象
未识别：3 个图层 / 20 个对象
白名单：2 个图层 / 8 个对象
```

### 未识别图层节点

未识别节点默认展开，因为这是最高风险区域。

未勾选“未识别图层归 0”时：

```text
未识别图层（3 层 / 20 对象，保持原样）
  UNKNOWN-A    12 对象    未识别且未命中白名单
  UNKNOWN-B     8 对象    未识别且未命中白名单
```

勾选“未识别图层归 0”时：

```text
未识别图层（3 层 / 20 对象，将归到 0 层）
  UNKNOWN-A -> 0    12 对象    未识别且启用归 0
  UNKNOWN-B -> 0     8 对象    未识别且启用归 0
```

### 迁移图层节点

迁移节点默认折叠。其子节点按目标图层分组。

目标图层组按对象数从多到少排序：

```text
将迁移图层（18 层 / 1120 对象）
  0-设备层（14 层 / 1102 对象）
    Visible (ISO) -> 0-设备层    694 对象    命中别名 "*VIS*"（通配匹配）
    MACHINE-VIS35 -> 0-设备层    207 对象    命中别名 "*VIS*"（通配匹配）
  3-文字层（3 层 / 18 对象）
```

每个目标图层组默认折叠。用户可以展开查看源图层明细。

### 白名单节点

白名单节点默认折叠：

```text
白名单图层（2 层 / 8 对象，保持原样）
  0    1 对象    命中白名单 "0"（全字匹配）
  A-图框    7 对象    命中白名单 "*图框*"（通配匹配）
```

## 排序规则

- 未识别明细按对象数从多到少排序。
- 迁移目标组按组内对象数从多到少排序。
- 同一目标组内的源图层按对象数从多到少排序。
- 白名单明细按对象数从多到少排序。

## 数据结构

继续使用 `LayerStandardPlan` 表示单个源图层计划。

新增轻量统计 helper：

- 计算计划列表的源图层数和对象总数。
- 按目标图层聚合迁移计划。
- 生成 `TreeNode` 文本。

这些 helper 应尽量放在 `Plugin.cs` 的共享区域，便于 PowerShell 反射测试验证。`LayerCommands.cs` 负责把生成的节点添加到 `TreeView`。

## 交互行为

- 打开预览时，摘要和未识别节点默认展开。
- 迁移图层和白名单节点默认折叠。
- 勾选或取消“未识别图层归 0”时，清空并重建 TreeView。
- 执行按钮仍按当前复选框状态执行。
- 取消按钮不修改图纸。

## 测试范围

PowerShell 测试覆盖：

- 迁移计划按目标图层聚合。
- 目标图层组按对象数降序排列。
- 源图层明细按对象数降序排列。
- 未识别节点在 fallback 关闭时显示“保持原样”。
- 未识别节点在 fallback 开启时显示“将归到 0 层”。
- 白名单节点显示命中原因。
- `LayerCommands.cs` 使用 `TreeView`，不再使用纯文本 `TextBox` 作为主预览控件。

## 不在本轮范围

本轮不做：

- 搜索框
- 复制报告
- 逐项取消某个源图层
- 右键菜单
- 多列表格
- 改变实际迁移逻辑

这些功能可以在 TreeView 稳定后继续迭代。
