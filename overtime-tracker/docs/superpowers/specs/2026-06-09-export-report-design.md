# 导出报表格式统一设计

## 背景

当前管理员导出已经包含姓名、日期、星期、上下班、类型、事由、范围外、工时字段；个人“我的记录”导出字段较少，缺少姓名、星期、事由、范围外标记和小时格式。管理者需要从报表里快速筛选范围外记录、核对事由，并查看汇总数据。

## 目标

本轮只负责统一 CSV 报表格式和补充汇总信息，不新增费用计算，不新增费率设置。

统一明细列：

```text
姓名,日期,星期,上班,下班,类型,事由,范围外,工时(分),工时(h)
```

## 行为设计

个人导出和管理员导出使用同一套 CSV 生成逻辑。个人导出只包含当前用户；管理员导出按现有人员和时间范围筛选。

每个成员每天一行明细：

- `姓名`：显示姓名，缺失时使用用户标识。
- `日期`：记录日期。
- `星期`：周日到周六。
- `上班`：当天第一条上班记录时间。
- `下班`：当天最后一条下班记录时间，缺失时留空。
- `类型`：工作日或休息日。
- `事由`：当天所有非空事由用 `; ` 合并。
- `范围外`：当天任一记录超范围则填 `是`，否则留空。
- `工时(分)`：当天计入的分钟数。
- `工时(h)`：如 `2h30m`、`45m`。

管理员导出在明细后追加汇总行，便于直接查看合计：

```text
汇总,,,,,,,范围外 N 天,总计分钟,总计小时
```

个人导出也追加同样汇总行，保持格式一致。

## 非目标

- 不新增费用列。
- 不新增统一费率设置。
- 不改变现有工时计算规则。
- 不改变后端原始记录接口。

## 测试策略

新增前端 CSV 生成测试，覆盖：

- 个人和管理员使用相同列头。
- 范围外记录在 `范围外` 列标记为 `是`。
- 事由被保留并合并。
- CSV 底部包含汇总行。
- 总分钟、总小时和范围外天数计算正确。

保留现有后端导出接口测试，确认 `out_of_range` 和 `note` 仍随原始记录返回。
