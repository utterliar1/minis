# CadToolkit 图层规范预览工具设计

## 目标

继续打磨 `CT_LAYERSTANDARD` 的执行前预览，让大图纸里的风险信息更容易聚焦，同时方便把完整预览报告复制给同事核对。同步把项目默认图层映射里的 `0-设备层` 改成用户本地正在使用的写法。

## 范围

本轮只改预览窗口、复制报告能力和默认配置文本。

保留不变：

- 图层统计范围
- 图层匹配规则
- 白名单、未识别归 `0`、删除空旧图层、ByLayer 的实际执行逻辑
- TreeView 的分组结构和排序规则
- 用户本地 `C:\CadToolkit\CadToolkit.ini` 的保护策略

新增：

- TreeView 预览筛选：`全部`、`未识别`、`将迁移`、`白名单`
- `复制报告` 按钮
- 默认配置同步：`0-设备层=*设备*,0-4,*VIS*`

## 预览筛选

在 TreeView 上方增加一组小按钮或单选式按钮：

```text
全部  未识别  将迁移  白名单
```

默认选中 `全部`。

筛选规则：

- `全部`：显示摘要、未识别图层、将迁移图层、白名单图层。
- `未识别`：显示摘要和未识别图层。
- `将迁移`：显示摘要和将迁移图层。
- `白名单`：显示摘要和白名单图层。

筛选只改变当前 TreeView 显示，不改变实际执行计划。即使当前只看 `未识别`，点击 `执行` 时仍按完整计划执行。

当用户勾选或取消 `未识别图层归 0` 时，当前筛选模式保持不变，只重建对应筛选下的树节点。未识别节点文案继续在 `保持原样` 和 `将归到 0 层` 之间切换。

## 复制报告

在底部按钮区域增加 `复制报告` 按钮。

行为：

- 点击后把完整文本报告复制到剪贴板。
- 复制内容使用完整报告，不受当前筛选模式影响。
- 报告内容可继续复用现有 `FormatLayerPlan(...)`。
- 复制成功后可用消息框或按钮附近提示告知 `已复制报告`。
- 如果剪贴板写入失败，显示错误提示，但不影响预览窗口和执行按钮。

## 默认配置同步

将项目内默认配置的 `0-设备层` 映射改为：

```ini
0-设备层=*设备*,0-4,*VIS*
```

同步位置：

- `CadToolkit/CadToolkit.ini`
- `CadToolkit/CadToolkit.default.ini`
- `CadToolkit/src/CadToolkit.Core/Config.cs` 内置默认文本
- `CadToolkit/README.md`
- `CadToolkit/CadToolkit使用手册.html`
- 相关 PowerShell 测试断言

仍然不覆盖用户本地 `C:\CadToolkit\CadToolkit.ini`。部署脚本继续保留已有用户配置。

## UI 布局

当前窗口大小可从 `560 x 510` 适度增加到 `620 x 540` 左右，为筛选按钮和复制按钮留出空间。

建议布局：

- 顶部：筛选按钮行
- 中部：TreeView，占据主要空间
- 底部：ByLayer、删除空旧图层、未识别归 0
- 右下：复制报告、执行、取消

按钮文字保持简短，避免占用过多宽度。

## 测试范围

PowerShell 测试覆盖：

- 新增筛选 helper 能按模式返回正确顶层节点。
- `全部` 包含四类节点。
- `未识别` 只包含摘要和未识别节点。
- `将迁移` 只包含摘要和迁移节点。
- `白名单` 只包含摘要和白名单节点。
- fallback 勾选状态变化时仍能影响未识别节点文案。
- `LayerCommands.cs` 包含筛选按钮和复制报告按钮。
- 默认配置、默认模板、内置默认、README、使用手册都同步为 `*VIS*`。
- 本地配置测试只验证用户配置保持显式通配写法，不要求部署覆盖。

## 不在本轮范围

本轮不做：

- 搜索框
- 逐项取消某个源图层
- 右键菜单
- 多列表格
- 修改迁移执行逻辑
- 覆盖用户本地配置

## 自检

- 没有占位项。
- 筛选功能明确为只影响显示，不影响执行。
- 复制报告明确为完整报告，不受筛选影响。
- 默认配置同步范围包含模板、内置默认和文档。
- 范围足够小，可以作为一个实施计划完成。
